# 新闻自动精编与发布 — 使用说明

面向编辑/运营人员，用于**手动拉取** AMP 稿件、AI 精编后**发布到 WordPress**。不涉及数据库或服务器配置。

---

## 一、打开系统

在浏览器访问前台地址（由管理员提供，一般为 `http://127.0.0.1:5173`）。

（截图：系统首页总览，左侧配置区 + 右侧批次与列表）

---

## 二、首次使用前的检查

在左侧 **「执行策略配置」** 中确认：

1. **发布模式** 选 **「手动发布」**（测试推荐；自动发布为定时任务，见文末说明）。
2. **新闻精编提示词 (Prompt)** 已填写（一般由管理员维护，可按需微调）。
3. **WordPress 发布目标** 已添加至少一个站点（名称、站点链接、登录方式、账号密码）。

点击 **「保存配置并更新任务」**，看到“配置已保存”提示。

（截图：左侧配置面板 — 发布模式、Prompt、WordPress 目标列表）

> 大模型 API Key 由管理员在后台配置；若采集时报错“未配置 LLM API Key”，请联系管理员。

---

## 三、单次拉取与精编（核心测试流程）

### 1. 设置时间范围

在左侧 **「单次拉取测试」** 中：

- **开始时间 / 结束时间**：默认最近 24 小时，可按需修改。
- 点击 **「开始采集与改写」**。

任务执行期间会显示“采集中，请稍候…”，请等待完成（时间与稿件数量、模型速度有关）。

（截图：单次拉取测试区域 — 时间选择与按钮）

### 2. 查看批次与统计

完成后右侧会生成新批次，顶部显示：

| 指标 | 含义 |
|------|------|
| **当前批次** | 本次任务编号 |
| **待发布** | 模型**采纳**、可进入发布流程的条数 |
| **已发布** | 本批次已成功发到 WP 的条数 |

（截图：顶部批次下拉框与三张统计卡片）

---

## 四、审核待发布稿件

切换到 **「本批次待审结果」** 标签。

每条记录左右对照：

- **左侧**：原始稿件（标题、配图、正文）
- **右侧**：AI 改写结果（分类、标题候选、精编正文）

底部 **判定理由** 说明采纳或不采纳原因。

- 状态为 **ready_to_publish**：已采纳，可发布。
- 状态为 **noise**：噪音/不采纳，**不会**被批量发布。

（截图：单条记录卡片 — 原文与 AI 改写对照）

---

## 五、发布到 WordPress

> 仅 **「本批次待审结果」** 支持发布操作；**「已发布历史」** 仅查看，避免重复发布。

### 方式 A：批量发布（推荐）

1. 勾选要发布的条目，或点击 **「全选本页」**（只选 `ready_to_publish` 状态）。
2. 点击右上角 **「批量发布」**，确认后开始。
3. 按钮会显示 **「发布中 xx%」**，完成后进度归零。
4. 每条底部出现平台标签，如 `能动:published`，点击可打开 WP 预览链接。

（截图：勾选多条 + 批量发布按钮 + 发布中状态）

### 方式 B：单条发布

在单条卡片底部点击 **「单条发布」**，仅发布该条。

（截图：单条发布按钮位置）

### 说明

- 发布内容为 **AI 精编稿**，标题取 **headline_1**；原稿中的 **特色图** 会一并上传为 WP 特色图。
- 按 **行业分类** 自动匹配对应 WordPress 站点（如「能动」→ 能动站）。
- 偶发 `failed` 多为目标站超时，可对该条再次单条发布；已成功的不建议在「已发布历史」里重复操作。

---

## 六、查看已发布记录

切换到 **「已发布历史」** 标签，可查看本批次已发稿件及各平台链接。

该页**无**勾选框、**无**批量/单条发布按钮。

（截图：已发布历史列表 — 无发布操作控件）

---

## 七、切换历史批次

右上角批次下拉框可选择以往批次，查看当时的待审/已发布记录。

（截图：批次选择下拉框）

---

## 八、自动发布模式（可选）

若改为 **「自动发布」** 并设置 **定时拉取与发布间隔**（1 / 3 / 12 / 24 小时），保存后系统将按间隔自动拉取、改写并发布，无需手动点采集。

测试阶段建议使用 **手动发布**，便于逐批核对。

（截图：自动发布模式与定时间隔下拉）

---

## 九、常见问题

| 现象 | 处理建议 |
|------|----------|
| 采集失败，提示未配置 API Key | 联系管理员配置大模型密钥 |
| 待发布数为 0 | 本批稿件可能被模型判为噪音，查看判定理由 |
| 批量发布按钮灰色且显示「发布中 0%」 | 刷新页面；若仍异常，联系技术支持 |
| 批量发布提示请先勾选 | 在「本批次待审结果」勾选 `ready_to_publish` 条目 |
| 同条既有 failed 又有 published | 多为重试成功，以 published 链接为准 |

---

## 十、推荐测试 checklist

- [ ] 手动发布模式已保存  
- [ ] WordPress 目标已配置  
- [ ] 单次拉取最近 24 小时成功  
- [ ] 待审列表有采纳稿件，改写内容可读  
- [ ] 批量发布 1～2 条成功，WP 可预览  
- [ ] 「已发布历史」可看到记录且无重复发布入口
